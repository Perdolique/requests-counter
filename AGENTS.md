## Testing Requirements

- After each completed task, always run relevant automated checks or tests if they exist.
- At minimum, run the most relevant verification for the changes you made, such as unit tests, type checks, or linting.
- If tests or checks cannot be run, say that explicitly in the final response.
- Write or update tests when it is necessary to safely cover new logic, bug fixes, important edge cases, or behavior that is easy to break.
- Do not add pointless tests for trivial markup-only or wording-only changes unless they provide real value.
